See it live ↓ GitHub → 
import { createDashboard } from '@loupe/loupe';

await createDashboard({
  source: { url: '/data/sales.csv', table: 'sales', namespace: 'demo' },
  filters: ['region', 'product'],
  metrics: [
    { label: 'Revenue',  sqlExpr: 'SUM(revenue)',  format: v => `$${v}` },
    { label: 'Units',    sqlExpr: 'SUM(units)',    format: v => String(v) },
    { label: 'Deals',    sqlExpr: 'COUNT(*)',      format: v => String(v) },
    { label: 'Avg Deal', sqlExpr: 'ROUND(SUM(revenue)/COUNT(*))', format: v => `$${v}` },
  ],
  table:  { columns: ['date', 'region', 'product', 'rep', 'revenue'] },
});

Booting DuckDB-WASM…

The Problem

You have a CSV, a JSON API, or a Parquet file. You want an interactive dashboard you can share with a link. Today that means spinning up a server, configuring a database, wiring up a frontend, and deploying it somewhere. Loupe skips all of that. Your data goes straight into an in-browser SQL engine. Filters, metrics, and tables render automatically. The result is a static site you can host anywhere.

What You Get

One npm install, one config object. Everything else is handled.

SQL in the browser

DuckDB-WASM loads your data into a real SQL engine. Write queries against CSV, JSON, or Parquet — no server needed.

Reactive filters

Declare which columns are filterable. Loupe renders dropdowns that re-query on change. State persists across reloads.

Computed metrics

Define SQL expressions for KPIs. They update automatically when filters change. Custom formatters included.

Off-thread compute

Run forecasts, aggregations, or transforms in a Web Worker pool. The main thread stays responsive.

Deploy as static files

The output is HTML + JS. Host it on GitHub Pages, S3, Netlify, or any static file server. No backend to maintain.

Works offline

Once loaded, everything runs locally. No round-trips, no API keys, no usage limits.

Connect Your Data

Point Loupe at any data source. Static files, REST APIs, or S3 buckets — it all works.

// Static file — CSV, JSON, or Parquet
source: { url: '/data/sales.csv', table: 'sales' }

// REST API with auth headers
source: {
  url: 'https://api.example.com/orders',
  table: 'orders',
  fetchOptions: { headers: { Authorization: 'Bearer <token>' } },
}

// S3 / public cloud bucket
source: { url: 'https://my-bucket.s3.amazonaws.com/data.parquet', table: 'events' }

// Custom connector — plug in any data source
import { connect } from '@loupe/connect';
connect.register('mydb', myCustomConnector);
